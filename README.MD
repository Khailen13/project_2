# Проект X

## Описание:
Ядро для интернет-магазина.
### Модуль base_product
#### Класс BaseProduct
Родительский класс для класса продуктов (Product).
В классе описана общая функциональность, которая должна быть у каждого продукта.

### Модуль mixin_info
#### Класс MixinInfo
Класс-миксин при создании объекта, то есть при работе метода __init__,
печатает в консоль информацию о том, от какого класса и с какими параметрами был создан объект.

Например:
````
Product('Продукт1', 'Описание продукта', 1200, 10)
````

### Модуль product
#### Класс Product
Класс для продуктов Product имеет следующие свойства:
* название (name),
* описание (description),
* цена (price),
* количество в наличии (quantity).

Атрибут цены является приватным - написаны геттеры и сеттеры.
В сеттере реализована проверка: в случае если цена равна или ниже нуля,
выводится сообщение в консоль “Цена не должна быть нулевая или отрицательная”, при этом старая цена не меняется.
В случае если цена товара понижается, то требуется подтверждение пользователем вручную через ввод _y_(значит yes) или 
_n_ (значит no) для согласия понизить цену или для отмены действия соответственно.

Класс-метод new_product принимает на вход параметры товара в словаре
и возвращает созданный объект класса Product. В случае если товар уже существует,
количество старого товара и нового складываются. При конфликте цен устанавливается более высокая цена.
Количество товаров не должно быть равно 0, иначе вызывается ошибка с сообщением
"Товар с нулевым количеством не может быть добавлен".

Добавлено строковое отображение в следующем виде:
```
Название продукта, 80 руб. Остаток: 15 шт.
```
Реализована возможность складывать продукты (одного класса). Например,
для товара _a_ с ценой 100 рублей и количеством на складе 10 и 
товара _b_ с ценой 200 рублей и количеством на складе 2 
результатом выполнения операции _a_ + _b_ является 100 × 10 + 200 × 2 = 1400.

### Модуль category
#### Класс Category
Класс для категории продуктов Product имеет следующие свойства:
* название (name),
* описание (description),
* список товаров категории (products),

а также атрибуты класса: 
* общее количество категорий продуктов (category_count),
* общее количество продуктов (product_count).

Для добавления товаров в категорию реализован специальный метод add_product().
Можно добавлять только объекты класса Product и его классов-наследников.

Геттер products выводит список товаров в виде строк в формате:
```
Название продукта, 80 руб. Остаток: 15 шт.
```
Добавлено строковое отображение в следующем виде:
````
Название категории, количество продуктов: 200 шт.
````
Метод middle_price подсчитывает средний ценник всех товаров.
Когда в категории нет товаров, возвращается ноль.

### Модуль category_iterator
#### Класс CategoryIterator

Вспомогательный класс, с помощью которого можно перебирать товары одной категории,
например в цикле for. Для этого новый класс должен принимать на вход объект класса категории
и производить итерацию по товарам, которые хранятся в данной категории.

### Модуль smartphone
#### Класс Smartphone
Является классом-наследником класса Product.
Помимо наследованных от Product свойств обладает следующими:
* производительность (efficiency),
* модель (model),
* объем встроенной памяти (memory),
* цвет (color).

### Модуль lawn_grass
#### Класс Lawngrass
Является классом-наследником класса Product.
Помимо наследованных от Product свойств обладает следующими:
* страна-производитель (country),
* срок прорастания (germination_period),
* цвет (color).
